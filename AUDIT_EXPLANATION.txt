                    WHAT IS "AUDIT" IN YOUR PROJECT?
================================================================================

AUDIT = Security Review and Testing of Your Application

There are TWO types of audit in your GameForum project:

================================================================================
1. AUDIT LOGGING (Tracking User Actions)
================================================================================

WHAT IT IS:
Your application automatically records all security-related events in a log.

WHERE IT'S IMPLEMENTED:
- File: backend/utils/auditLogger.js
- Database: AuditLogs collection in MongoDB
- Figure 16 in documentation shows the code

WHAT IT LOGS:
✓ User login attempts (success/failed)
✓ User registration
✓ Password changes
✓ Admin actions (ban/unban users)
✓ MFA verification attempts
✓ Account lockouts
✓ Unauthorized access attempts

INFORMATION CAPTURED:
- User ID and username
- Action performed
- Timestamp
- IP address
- User agent (browser info)
- Status (success/failed)
- Additional metadata

WHY IT'S IMPORTANT:
- Accountability: Track who did what and when
- Security monitoring: Detect suspicious activity
- Forensics: Investigate security incidents
- Compliance: Meet regulatory requirements

EXAMPLE FROM YOUR CODE:
When a user logs in, the system logs:
- Username: john_doe
- Action: USER_LOGIN
- Timestamp: 2026-01-26 10:30:45
- IP: 192.168.1.100
- Status: SUCCESS

================================================================================
2. SECURITY AUDIT (Code Review & Penetration Testing)
================================================================================

WHAT IT IS:
You manually reviewed your code and tested it for security vulnerabilities.

TWO TYPES:

A) MANUAL CODE REVIEW
   - You read through your code looking for security flaws
   - Checked authentication, authorization, input validation
   - Verified security controls are properly implemented

B) INTERNAL PENETRATION TESTING
   - You tried to hack your own application
   - Discovered vulnerabilities by testing attack scenarios
   - Fixed the vulnerabilities you found

WHERE IT'S DOCUMENTED:
Section 10: "Vulnerability Discovery and Mitigation"

VULNERABILITIES YOU FOUND AND FIXED:

1. Weak Password Storage
   PROBLEM: Considered storing plain passwords
   FIX: Implemented bcrypt hashing with 12 salt rounds

2. Brute Force Attacks
   PROBLEM: Unlimited login attempts
   FIX: Rate limiting (10 attempts/15 min) + account lockout

3. Session Hijacking
   PROBLEM: JWT in localStorage (vulnerable to XSS)
   FIX: Moved to HTTP-only cookies

4. Unauthorized Access
   PROBLEM: Any user could access admin functions
   FIX: Implemented RBAC middleware

5. Account Takeover
   PROBLEM: Banned users could still login
   FIX: Unlock accounts when unbanning

TESTING PERFORMED:
✓ 75 automated unit tests (100% pass rate)
✓ Manual penetration testing
✓ Code review for security flaws
✓ Vulnerability scanning
